# Lumin AI — لومين

مساعد ذكاء اصطناعي محادثاتي، من تطوير **محمد محسن** (البصرة، مواليد 2008)،
رئيس **مجموعة شركات لومين انفينيتي**.

- موقع المطوّر: https://muhamed1.netlify.app
- موقع الشركة: https://lumin-infinity.netlify.app

---

## 1. كيف يعمل المفتاح الآن؟

المفتاح **مدمج تلقائيًا** داخل `netlify/functions/chat.js` (الثابت `EMBEDDED_FALLBACK_KEY`)،
فالموقع يعمل فورًا بعد النشر دون أي إعداد إضافي على لوحة Netlify.

- الواجهة (`index.html`, `style.css`, `app.js`) لا تحتوي على المفتاح أبدًا ولا
  يصل إلى متصفح أي زائر — فهو موجود فقط داخل دالة الخادم (`chat.js`) التي
  تُنفَّذ على خوادم Netlify ولا تُشحن للمتصفح إطلاقًا (لن تجده عبر F12 →
  Network/Sources مهما حاول أحد).
- لو ضبطت متغيّر بيئة `OPENROUTER_API_KEY` من لوحة Netlify، فإنه يأخذ
  الأولوية تلقائيًا على المفتاح المدمج، دون حاجة لتعديل الكود.

⚠️ **تنبيه مهم:** "لا يظهر في كود الموقع" يعني أنه لا يظهر لزوّار الموقع.
لكنه يبقى مكتوبًا بصريح العبارة داخل ملف `chat.js` نفسه. إن رفعت هذا المشروع
على مستودع GitHub **عام**، سيتمكن أي شخص من رؤية المفتاح في كود المستودع
ونسخه. لتفادي ذلك:
- اجعل المستودع **خاصًا (Private)**، أو
- احذف السطر الذي يحتوي المفتاح من `chat.js` بعد رفع المشروع واعتمد فقط على
  متغيّر البيئة `OPENROUTER_API_KEY` من لوحة Netlify (هذه كانت الطريقة
  الأصلية الأكثر أمانًا، وما زالت تعمل تلقائيًا بمجرد ضبطها).

## 2. خطوات النشر على Netlify

1. أنشئ مستودعًا (GitHub/GitLab) يحتوي على كل ملفات هذا المشروع، أو اسحب
   المجلد كاملًا مباشرة في صفحة "Deploy manually" على Netlify.
2. هذه الخطوة **اختيارية الآن** لأن المفتاح مدمج مسبقًا، لكنها تبقى الأفضل
   أمنيًا: من لوحة تحكم الموقع في Netlify: **Site configuration →
   Environment variables** → أضف متغيّرًا:
   - الاسم: `OPENROUTER_API_KEY`
   - القيمة: مفتاحك من https://openrouter.ai/keys
   - (اختياري) أضف أيضًا `LUMIN_MODEL` لو رغبت بتغيير النموذج المستخدم، مثل
     `openai/gpt-4o-mini` أو `anthropic/claude-3.5-sonnet` أو أي نموذج تجده
     مناسبًا في https://openrouter.ai/models (الأسماء والتوافر يتغيّر بمرور
     الوقت، راجع الصفحة دائمًا قبل الاختيار).
3. أعد نشر الموقع (Trigger deploy) بعد إضافة المتغيّر.
4. افتح الموقع، ستلاحظ النقطة الخضراء بجانب "متصل بخادم لومين" في الأعلى.

## 3. تجربة سريعة محليًا (اختياري، لمن يملك Node.js)

```bash
npm install -g netlify-cli
netlify dev
```

سيشغّل هذا الأمر الموقع والدالة معًا محليًا على نفس المنفذ، مع قراءة
متغيّرات البيئة من ملف `.env` إن أنشأته يدويًا (لا تضع هذا الملف في أي
مستودع عام).

## 4. تخصيص شخصية لومين

كل معلومات الهوية والتلقين موجودة في مكان واحد فقط:
`netlify/functions/chat.js` → الثابت `LUMIN_SYSTEM_PROMPT`.
عدّل النص هناك لإضافة أو تحديث أي معلومة، دون الحاجة لتعديل أي ملف آخر.

## 5. هيكل الملفات

```
lumin-ai/
├── index.html                 الواجهة وبنية الصفحة
├── style.css                  الهوية البصرية (تصميم متوهّج أخضر/أسود)
├── app.js                     منطق المحادثة، التخزين، إعادة المحاولة، الإعدادات
├── netlify.toml                إعداد مسار /api/chat
├── netlify/functions/chat.js   الدالة الآمنة + هوية لومين + الاتصال بـ OpenRouter
└── README.md                   هذا الملف
```

## 6. ملاحظة عن البث الفوري (Streaming)

تستخدم النسخة الحالية طلبًا عاديًا (غير مُجزَّأ) إلى OpenRouter، مع تأثير
"كتابة تدريجية سريعة" في الواجهة لإحساس استجابة فورية وسريع — وهذا يضمن
توافقًا كاملًا مع أي حساب Netilfy عادي دون أي إعداد إضافي. إن رغبت لاحقًا
في بثّ حقيقي حرفًا بحرف من النموذج نفسه، يمكن تطوير الدالة لاستخدام ميزات
الاستجابة المتدفقة (Streaming Responses) في Netlify Functions — راجع وثائق
Netlify الرسمية لأحدث طريقة معتمدة عند رغبتك بهذا التحديث.
